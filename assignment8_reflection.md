
Reflection

In this assignment, I learned how to model the behavior of a system using state diagrams and activity diagrams.

One challenge I faced was deciding how much detail to include in the diagrams. If I added too much detail, the diagrams became confusing. If I added too little, they were not useful. I had to find a balance between clarity and detail.

Another challenge was making sure the diagrams matched my previous assignments, especially the user stories and use cases. I had to carefully check that everything was consistent.

I also learned the difference between state diagrams and activity diagrams. State diagrams focus on how an object changes over time, while activity diagrams show the steps in a process.

Overall, this assignment helped me understand how to visualize system behavior and workflows. It is an important step before building the actual system.
